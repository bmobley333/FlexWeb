🏛️ System Purpose & User Workflows
The FlexMoxie - CS is the active player-facing user interface and combat dashboard. It translates character statistics, attributes, equipment, and abilities into an interactive, automated roll engine. Rather than querying remote databases dynamically on user clicks (which causes performance latency), the CS template implements local cache worksheets that store filtered subsets of powers, magic items, and skillsets. This local cache is then queried instantly by data validations and bound triggers.

⚙️ Bound Script Entry Points (CS/Code.js)
The character sheet script CS/Code.js directs events to FlexLib:
1. Simple Trigger: onOpen()
* If the sheet is initialized (PropertiesService.getScriptProperties()), it creates the player menus (FlexLib.fCreateFlexMenu()). If the current user is the administrator (metascapegame@gmail.com), it also appends FlexLib.fCreateDesignerMenu('CS').
* If not initialized, it shows the activation menu pointing to fActivateMenus().
2. Sheet Initialization Trigger: fActivateMenus()
* Executes FlexLib.run('CharacterOnboarding') to sync libraries, check core database checkboxes, run initial data filters, and set the default tab to Game.
3. Active Edit Listener: onEdit(e)
Monitors edits on the Game tab to automate detail population and skillset updates: * Dropdown Syncing (Powers & Items): * Intercepts choices made in powerdropdown1/2 or magicitemdropdown1/2. * Instantly clears existing fields and queries the local caches (PowerDataCache/MagicItemDataCache). * Auto-populates the corresponding usage, action, name, and effect columns. * Skillset Modification & Auto-Placement: * Intercepts selections in skillsetdropdown. * Removal Step: If a skillset was modified/cleared, reads the old skillset's skills list (Column skillseteffect), parses them, and executes fUpdateCharacterSkills(sheet, skills, gameColTags, 'REMOVE'). This locates the Might/Motion/Mind/Magic/Moxie skills rows on the Game tab and decrements or removes each skill (e.g., removing Acrobatics🏃). * Addition Step: If a new skillset is selected, reads its skill list from SkillSetDataCache and runs fUpdateCharacterSkills(..., 'ADD') to write the new skills to the appropriate attribute row, incrementing counts (e.g. 2_Focus👁️) if duplicates exist.
4. Installable Structural Trigger: onChange(e)
* Monitors structural changes (INSERT_ROW, REMOVE_ROW, INSERT_COLUMN, REMOVE_COLUMN) on the Game tab.
* Instantly calls FlexLib.run('InvalidateGameCache') to wipe cached row and column indices. This protects the coordinates system from drifting if rows or columns are added.

🔀 Core Logic Workflows (Codebase Analysis)
I. Skill List Incrementor & Decrementor (fUpdateCharacterSkills)
Dynamically manages the comma-separated skill rows on the Game tab: 1. Attribute Mapping: Reads the ending emoji (e.g., 💪) of a skill and maps it to its target worksheet row tag (e.g. mightskills). 2. Double-Row Placement: Checks two consecutive rows for that attribute (e.g., Might Row 1 and Might Row 2). 3. Execution: * ADD Mode: Searches if the skill name already exists. If found, splits the count prefix (e.g., 2_Climb💪 ➔ increments count to 3_Climb💪). If new, adds it to the shorter of the two rows to distribute visual density. * REMOVE Mode: Searches for the skill. If found with a count prefix, decrements the count. If no prefix remains, removes the skill string entirely from the row.
II. Print-Prep & Visual Cleanup (fPrepGameForPaper)
1. Resetting Paper: Deletes the existing Paper tab.
2. Duplication: Clones the Game tab as a new worksheet named Paper and positions it at index 1.
3. Designer Strip: Reads the Cell A1 note of the Game sheet (which contains the configuration string Hide: [A1-ranges]).
4. Deletions: Converts the notation string into column and row indices, sorts them in reverse order, and deletes the designer-only layout blocks from Paper to leave the player with a clean character sheet.
