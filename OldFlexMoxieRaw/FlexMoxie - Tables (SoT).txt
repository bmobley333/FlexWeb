🏛️ System Purpose & User Workflows
The FlexMoxie - Tables is the master rules schema and statistics database. It houses the system-wide progression charts, gear catalogs, combat styles, class attributes, monster stats, and skill groups. The spreadsheet serves as the numeric database for character calculations. Additionally, it contains historical/backup worksheets for player characters, serving as a GM archives ledger.

🔀 Core Logic Workflows (Codebase Analysis)
I. Skill Emoji Verification Workflow (fVerifyIndividualSkills)
Ensures that all skills in the Skills worksheet follow the correct string formatting: 1. Verification Checks: Verifies the worksheet is tagged with a header row note and the targeted column has a skills tag. 2. String Scanning: Loop through cells in the skills column. Calls the validation helper: * Capitalizes all words in the skill name (e.g., pick pocket ➔ Pick Pocket). * Scans for the five core attributes: 💪, 🏃, 👁️, ✨, and 🫀. * Auto-Correction: If exactly one emoji is present, the script repositions the emoji to the end of the text string (e.g. Acrobatics🏃). * Resolution Prompt: If zero or multiple emojis are found, the script opens a dialog prompting the user to select the correct attribute index (1–5) and auto-corrects the cell.
II. SkillSet Comma-Separated List Verification (fVerifySkillSetLists)
Verifies that comma-separated lists of skills within the SkillSets worksheet are formatted correctly: 1. Target Validation: Halts early if the active worksheet is not SkillSets. Verifies tags: header (row), skillset (column), and skilllist (column). 2. List Split Scan: Splits the comma-separated strings in the skilllist column into individual skills. 3. Array Loop Correction: Feeds each split skill to fValidateAndCorrectSkillString() to format emojis and capitalization, re-combines the array into a clean comma-separated list (e.g. Athletics💪, Acrobatics🏃, Focus👁️), and updates the cell if modifications occurred.
